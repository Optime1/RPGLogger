# БИЗНЕС-ПРОЦЕСС: Логирование сессий НРИ


## КЛЮЧЕВОЙ ПРОЦЕСС

| Параметр | Описание |
|----------|----------|
| **Название процесса** | Логирование и ведение записей по сессиям НРИ |
| **Владелец процесса** | Мастер игры (ГМ/DM) |
| **Участники процесса** | Мастер, Игроки |
| **Триггер начала** | Начало игровой сессии |
| **Триггер окончания** | Экспорт логов игрокам после сессии |
| **Частота выполнения** | 1 раз в 1-2 недели (типичная периодичность сессий) |
| **Входные данные** | События сессии, действия игроков, решения ГМа |
| **Выходные данные** | Структурированный лог сессии, прогресс персонажей |


## AS-IS ПРОЦЕСС (Текущее состояние)

### Схема процесса до внедрения приложения

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        AS-IS: ТЕКУЩИЙ ПРОЦЕСС                           │
└─────────────────────────────────────────────────────────────────────────┘

  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
  │   Начало     │────▶│  Ведение     │───▶│  Завершение  │
  │   сессии     │     │  записей     │     │   сессии     │
  └──────────────┘     └──────────────┘     └──────────────┘
                              │                      │
                              ▼                      ▼
                       ┌──────────────┐     ┌──────────────┐
                       │   Бумага/    │     │   Отправка   │
                       │   Заметки/   │     │   игрокам    │
                       │   Discord    │     │ (если вообще)│
                       └──────────────┘     └──────────────┘
```

### Детальное описание шагов AS-IS

| № | Шаг | Действие | Инструмент | Время | Ответственный |
|---|-----|----------|------------|-------|---------------|
| 1 | Подготовка | Подготовка записей к сессии | Бумага/Блокнот | 5-10 мин | Мастер |
| 2 | Логирование | Запись событий во время игры | Бумага/Заметки в телефоне | В процессе игры | Мастер |
| 3 | Структурирование | Приведение записей в порядок после сессии | Текстовый редактор | 30-60 мин | Мастер |
| 4 | Хранение | Сохранение записей | Google Docs/Бумага/Notion | 5 мин | Мастер |
| 5 | Распространение | Отправка логов игрокам | Discord/Email/Мессенджер | 10-15 мин | Мастер |
| 6 | Доступ игроков | Игроки получают и читают логи | Разные устройства | Зависит от игрока | Игроки |

### Участники процесса AS-IS

| Роль | Обязанности | Боли |
|------|-------------|------|
| **Мастер** | Ведение записей, структурирование, отправка | Тратит много времени, записи теряются, сложно искать |
| **Игрок** | Получение логов, ведение личных заметок | Не всегда получает логи, нет единого формата, сложно отслеживать прогресс |


## АНАЛИЗ AS-IS: ПРОБЛЕМЫ И НЕУДОБСТВА

### Проблемы

| № | Проблема | Описание | Влияние | Частота |
|---|----------|----------|---------|---------|
| 1 | **Потеря данных** | Бумажные записи теряются, файлы удаляются | Потеря истории кампании | Высокая |
| 2 | **Отсутствие структуры** | Записи в произвольном формате | Сложно искать информацию | Всегда |
| 3 | **Временные затраты** | Мастер тратит 30-60 мин после сессии на оформление | Выгорание Мастера | Каждая сессия |
| 4 | **Нет синхронизации** | Игроки не получают логи вовремя | Разное понимание сюжета | 70% сессий |
| 5 | **Сложность поиска** | Нет индексации событий | Невозможно быстро найти прошлое событие | При необходимости |
| 6 | **Разные инструменты** | У каждого игрока свой способ ведения заметок | Несогласованность данных | Всегда |
| 7 | **Нет бэкапа** | Данные не резервируются | Риск полной потери кампании | Средняя |
| 8 | **Сложность импорта** | При смене устройства данные не переносятся | Потеря прогресса | При необходимости |



## TO-BE ПРОЦЕСС (После внедрения приложения)

### Схема процесса с приложением

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    TO-BE: ПРОЦЕСС С ПРИЛОЖЕНИЕМ                         │
└─────────────────────────────────────────────────────────────────────────┘

  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
  │   Начало     │────▶│  Логирование │────▶│  Завершение  │
  │   сессии     │     │  тегами      │     │   сессии     │
  └──────────────┘     └──────────────┘     └──────────────┘
                              │                      │
                              ▼                      ▼
                       ┌──────────────┐     ┌──────────────┐
                       │   Авто-      │     │   Экспорт    │
                       │   сохранение │     │   в 1 клик   │
                       │   в БД       │     │   игрокам    │
                       └──────────────┘     └──────────────┘
                              │                      │
                              ▼                      ▼
                       ┌──────────────┐     ┌──────────────┐
                       │   Поиск и    │     │   Импорт     │
                       │   фильтрация │     │   игроками   │
                       └──────────────┘     └──────────────┘
```

### Детальное описание шагов TO-BE

| № | Шаг | Действие | Инструмент | Время | Ответственный |
|---|-----|----------|------------|-------|---------------|
| 1 | Подготовка | Открытие кампании в приложении | NRI Logger App | 1 мин | Мастер |
| 2 | Логирование | Клик по тегу во время события | NRI Logger App | 2-5 сек на событие | Мастер |
| 3 | Завершение | Нажатие "Завершить сессию" | NRI Logger App | 1 мин | Мастер |
| 4 | Экспорт | Выбор тегов и экспорт | NRI Logger App | 7-10 мин | Мастер |
| 5 | Распространение | Отправка файла игрокам | Мессенджер | 2 мин | Мастер |
| 6 | Импорт | Игроки импортируют сессию | NRI Logger App | 1 мин | Игроки |
| 7 | Просмотр | Игроки читают логи в любое время | NRI Logger App | По необходимости | Игроки |

### Участники процесса TO-BE

| Роль | Обязанности | Преимущества |
|------|-------------|--------------|
| **Мастер** | Клик по тегам, экспорт | Экономия 40+ мин на сессию, структурированные данные, авто-бэкап |
| **Игрок** | Импорт сессии, просмотр | Всегда актуальные логи, поиск по истории, отслеживание прогресса |

### Сравнение AS-IS vs TO-BE

| Параметр | AS-IS | TO-BE | Улучшение |
|----------|-------|-------|-----------|
| Время на логирование сессии | 45-60 мин | 5-10 мин | **80-85% экономия** |
| Структура данных | Отсутствует | Стандартизированная | **100% улучшение** |
| Поиск информации | Ручной, долгий | Мгновенный, по фильтрам | **90% экономия времени** |
| Синхронизация с игроками | 30% сессий | 95% сессий | **3x улучшение** |
| Риск потери данных | Высокий | Низкий (авто-сохранение) | **80% снижение риска** |
| Доступ к истории | Ограниченный | Полный, с поиском | **100% улучшение** |
| Бэкап данных | Ручной, редко | Автоматический | **100% улучшение** |
